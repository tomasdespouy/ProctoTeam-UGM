# **App Name**: UGM Proctor

## Core Features:

- Student Exam Interface: Student view with exam questions, timer, and video feed.
- Instructor Monitoring Dashboard: Instructor dashboard to view live feeds, alerts, and student progress.
- Behavior Detection: Real-time monitoring for suspicious behavior like tab switching or audio spikes.
- AI-Powered Proctoring Alerts: Use a tool to determine whether flagged events during the exam warrant escalation to a proctor, and provide supporting information for the proctor's review.
- Emergency Support: Emergency support button for students to request assistance during the exam.
- Exam Recording: Record exam sessions to allow later review. Recording indicator shown on the screen.
- Detailed Alert System: Clear visual alerts with tooltips explaining the detected events during the examination process, and the system's evaluation of them.

## Style Guidelines:

- Primary color: Dark blue (#002366) to reflect the university's brand identity and seriousness.
- Background color: Light gray (#F8F9FA) for the student view and a slightly darker gray (#F0F2F5) for the instructor dashboard, providing a clean and neutral backdrop.
- Accent color: Gold (#FFD700) for the 'Soporte técnico' button and important alerts, drawing user attention.
- Body font: 'PT Sans' sans-serif font for the main text to provide a clear and readable experience. 'Space Grotesk' will be used for headings.
- Note: currently only Google Fonts are supported.
- Utilize Material Design Icons for a consistent and recognizable visual language.
- Split the student view into two columns: one for the exam questions (70% width) and another for the video feed and alerts (30% width). Instructor view employs a grid layout for student thumbnails and a sidebar for alerts.
- Subtle transition effects (0.3s ease-in-out) for button hovers and alert updates, enhancing user interaction.
